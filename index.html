Feature #1 — חדש

📢 תקשורת יזומה + מעקב אישורים

מנהל יוצר "עדכון לעובדים" — נוהל חדש, מנה חדשה, שינוי תפריט. המערכת שולחת הודעת WhatsApp לכל עובדים רלוונטיים עם כפתור אישור קריאה. הדשבורד מראה בזמן אמת מי אישר ומי לא.

  • עובד מקבל הודעה ישירה, לא "יש לכם הודעות חדשות"
  • אישור קריאה בלחיצה אחת — אין ניווט לאף מקום
  • מנהל רואה מי לא קרא → פולואופ ממוקד
  • מתועד: מי קרא, מתי — חוקי ותאגידי
  • תזכורת אוטומטית לאחר X שעות למי שלא אישר
  • סגמנטציה: לכל העובדים / לסניף ספציפי / לתפקיד

05 — ארכיטקטורה

המודל ההיברידי

לא "WhatsApp במקום UI" — כל כלי למשתמש הנכון.

💬

WhatsApp — עובדי חזית

דיווחים, קבלת עדכונים, אישורים, שאילתות מהירות. פועל בזמן אמת במהלך המשמרת. אפס חיכוך.

🖥️

Dashboard UI — מנהלים

דשבורד עדכונים, אנליטיקה, קונפיגורציה של תבניות, צפייה בלוגים ואישורים. מתאים לסשן ממושך.

⚙️

Base44 Backend

State machine, entities, functions. פריסה אוטומטית על git push. הבוט רץ על Deno Serverless.

📡

99Digital API

WhatsApp Business API ישראלי. Webhook inbound, sendTemplate לסשנים חדשים, sendMessage בתוך 24 שעות.

📢

Broadcast Engine (חדש)

שליחת הודעות יזומות לקבוצות עובדים. מעקב ack per-employee. תזכורות אוטומטיות ל-pending.

📰

News Feed Engine (חדש)

Entity: EmployeeBroadcast + per-employee seen tracking. על כניסה לבוט — בדיקת unseen items לפי חומרה.

06 — סיכום

המסקנה: Bot-First, UI-Second

עובדי חזית לא ישתמשו בממשק ארגוני — הנתונים ברורים. WhatsApp מנצח על חיכוך, על מוכרות, ועל זמינות.

האסטרטגיה הנכונה: Bot לכל פעולה שעובד צריך לבצע, UI לכל מה שמנהל צריך לנהל. זה לא בחירה — זה שכבות.
